README

The basis for the subfolder fmu/ext was taken from

https://github.com/open-simulation-platform/cpp-fmus

which is published via MIT license.

Therefore, we are good to modify it and use it in our proprietary code.

Some parts of the original project were deleted as these were examples and are not needed here.

Also, the submodule reference to https://github.com/viproma/cppfmu was removed in order to not run

into any dependencies there.

The CMakeList was adapted to fit to this project.

Build up:

  • The "ext" folder mainly contains the sources to bind the fmi sources as defined by the fmi standard. It also contains the CMakeList.
  • The "long_control" folder contains the logic that is used inside the fmu.
  • The "fmu" folder contains the packed fmu. This is the compiled dll as well as the modelDescription.xml that are zipped and renamed. Thus, the .fmu

file can still be opened with an archive program like 7z.

The build instructions are described in detail in the readme inside the "ext" folder.

E.g. for building 64-bit you can do the following:

cd ext

mkdir build

cd build

cmake -G "Visual Studio 15 2017" -A x64 -S ..

cmake –build .

When you want to clean the target and rebuild you can execute

cmake –build . –clean-first

This also gives you a newly compiled fmu in the "fmu" folder.
