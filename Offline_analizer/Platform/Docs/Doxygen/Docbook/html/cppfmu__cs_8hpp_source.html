120 /* A function which must be defined by model code, and which should create
121  * and return a new slave instance.
122  *
123  * The returned instance must be managed by a std::unique_ptr with a deleter
124  * of type std::function<void(void*)> that takes care of freeing the memory.
125  * The simplest way to set this up is to use cppfmu::AllocateUnique() to
126  * create the slave instance.
127  *
128  * Most of its parameters correspond to those of fmi2Instantiate() and
129  * fmiInstantiateSlave(), except that 'functions' and 'loggingOn' have been
130  * replaced with more convenient types:
131  *
132  * memory = An object which the model code can use for memory management,
133  * typically in conjunction with cppfmu::Allocator,
134  * cppfmu::AllocateUnique(), etc. Allocation and deallocation
135  * requests get forwarded to the simulation environment.
136  *
137  * logger = An object which the model code can use to log messages (e.g.
138  * warnings or debug info). The messages are forwarded to the
139  * simulation environment's logging facilities.
140  *
141  * Note that this function is declared in the global namespace.
142  */
