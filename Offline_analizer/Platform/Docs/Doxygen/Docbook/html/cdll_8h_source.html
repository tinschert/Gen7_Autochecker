247 /*------------------------------------------------------------------------
248 
249 Since CANoe / CANalyzer 6.0 SP2, you may export the DLL function tables
250 as variables instead of using caplDllGetTable5 / caplDllGetCallbackTable5.
251 
252 You should do that if the compiler shall not attempt to load additional
253 modules referenced by your DLL or to execute code in your DLL. Examples are
254 DLLs for Windows CE or DLLs which need special driver DLLs. Technically, the
255 compiler will load the DLL using the Windows function LoadLibraryEx and the
256 parameter DONT_RESOLVE_REFERENCES.
257 
258 Export your function table with the name caplDllTable5/4/3/2 and your callback table
259 with the name caplDllCallbackTable5/4/3/2.
260 
261 Example (in the implementation file):
262 
263 CAPL_DLL_INFO5 table[] = {
264 
265 {CDLL_VERSION_NAME, (CAPL_FARCALL)CDLL_VERSION, "", "", CAPL_DLL_CDECL, 0xabcd, CDLL_EXPORT },
266 
267 {"HelloWorld", (CAPL_FARCALL) HelloWorld, CAPL_CONTEXT_ALL, "Hello World", "Copies 'Hello, World!' to a buffer",
268 'L', 2, "CD", "\001\000", {"buffer", "bufferSize"} },
269 
270 {0, 0}
271 };
272 CAPLEXPORT CAPL_DLL_INFO5 far * caplDllTable5 = table;
273 
274 Be careful to define all elements of the table statically, i.e., none of them
275 may be calculated at runtime. In particular, do not use other runtime-initialized
276 variables (e.g. variables of type char*) in the table definition.
277 
278 If the variable caplDllTable5 is not found, the compiler will reload the DLL
279 using LoadLibrary and call the exported functions if the library is for Win32.
280 If the library is for Windows CE, the compiler will call the exported functions
281 without reloading the DLL.
282 
283 ----------------------------------------------------------------------------*/
