Reading FDX values

Calculation_of() number of detected VALID objects that have been sent to the bus

  • Checking if the dx changed compared to the last cycle is used to identify if Cloe is still updating it and therefor if it is a valid object
  • FDX does not update values as often as Classe which is what we have to balance out by counting objects as valid as long as the values change every 5 cycles

Only update values if the object is in the FOV AND the valid flag is set AND it is a valid object
